comandos de navegación

pwd
pwd
Print Working Directory
básico ▶

Muestra la ruta absoluta del directorio en el que te encuentras actualmente. Siempre úsalo para saber dónde estás antes de crear archivos.

ejemplo
$ pwd
/home/seba

$ pwd
/etc/ansible
Tip: Si alguna vez estás perdido en la terminal, escribe pwd para orientarte.
ls
ls
Listar contenido de un directorio
básico ▶

Lista los archivos y carpetas del directorio actual o del que le indiques. Con opciones puedes ver detalles, archivos ocultos y más.

uso básico
$ ls                   # lista simple
$ ls -l               # formato largo (permisos, tamaño)
$ ls -la              # incluye archivos ocultos
$ ls -lh              # tamaño legible (KB, MB)
$ ls /etc/ansible     # lista otro directorio
salida de ls -la
total 20
drwxr-xr-x. 4 seba seba  68 abr 10 09:00 .
drwxr-xr-x. 3 root root  20 abr 10 08:00 ..
drwxr-xr-x. 2 seba seba   6 abr 10 09:00 inventario
drwxr-xr-x. 2 seba seba  40 abr 10 09:00 playbooks
-rw-r--r--. 1 seba seba 230 abr 10 09:00 ansible.cfg
Tip: La primera columna muestra los permisos. d al inicio significa directorio.
cd
cd
Change Directory
básico ▶

Navega entre directorios. Es el comando que más usarás en la terminal.

variantes de cd
$ cd /home/seba         # ruta absoluta
$ cd ansible-lab       # ruta relativa
$ cd ..               # sube un nivel
$ cd ../..            # sube dos niveles
$ cd ~                # va a tu home
$ cd -                # regresa al directorio anterior
Tip: Presiona Tab para autocompletar rutas. Presiona dos veces para ver opciones disponibles.

gestión de archivos y directorios

mkdir
mkdir
Make Directory — crear carpetas
creación ▶

Crea directorios. La opción -p es fundamental para Ansible: crea toda la ruta aunque las carpetas intermedias no existan, y no falla si ya existe.

ejemplos
$ mkdir proyecto

# -p: crea toda la ruta de una vez
$ mkdir -p ansible-lab/inventario
$ mkdir -p ansible-lab/{inventario,playbooks,roles,group_vars}

# estructura completa de un rol
$ mkdir -p roles/webserver/{tasks,handlers,templates,files,vars,defaults,meta}
Tip: La sintaxis {a,b,c} (brace expansion) crea múltiples carpetas en un solo comando.
touch
touch
Crear archivos vacíos
creación ▶

Crea un archivo vacío si no existe, o actualiza la fecha de modificación si ya existe. Ideal para crear los archivos YAML de Ansible antes de editarlos.

ejemplos
$ touch inventario/hosts
$ touch playbooks/{site,setup,usuarios,db}.yml
$ touch roles/webserver/tasks/main.yml
echo / cat
echo & cat
Escribir y leer contenido
contenido ▶

echo imprime texto. Con > sobreescribe un archivo y con >> agrega al final. cat muestra el contenido de un archivo.

echo con redirección
$ echo "[webservers]" > inventario/hosts
$ echo "192.168.1.10" >> inventario/hosts
$ cat inventario/hosts
[webservers]
192.168.1.10
heredoc — escribir varias líneas
$ cat > inventario/hosts << 'EOF'
[webservers]
192.168.1.10
192.168.1.11

[dbservers]
192.168.1.20
EOF
Tip: El heredoc << 'EOF' es la forma más rápida de escribir bloques de texto en un archivo desde la terminal.
nano
nano
Editor de texto en terminal
edición ▶

Editor de texto interactivo dentro de la terminal. Ideal para editar archivos YAML de Ansible. Es más amigable que Vim para comenzar.

abrir y atajos
$ nano playbooks/site.yml

  Ctrl + S  →  guardar
  Ctrl + X  →  salir
  Ctrl + K  →  cortar línea
  Ctrl + W  →  buscar texto
  Ctrl + /  →  ir a línea número
cp / mv
cp & mv
Copiar y mover archivos
gestión ▶

cp copia archivos o directorios. mv los mueve o renombra. Usa -r con cp para copiar directorios completos.

cp — copiar
$ cp ansible.cfg ansible.cfg.bak           # backup
$ cp -r roles/webserver roles/appserver  # copiar dir
$ cp ~/plantilla.yml playbooks/nuevo.yml   # desde otro dir
mv — mover/renombrar
$ mv setup.yml instalacion.yml        # renombrar
$ mv ~/hosts inventario/hosts         # mover archivo
$ mv roles/old /opt/backups/roles/    # mover directorio
rm
rm
Remove — eliminar archivos
¡cuidado! ▶

Elimina archivos y directorios. En Linux no hay papelera desde terminal — lo que borras, se va.

uso seguro
$ rm archivo.yml             # eliminar archivo
$ rm -i archivo.yml          # pide confirmación (-i es seguro)
$ rm -r roles/backup/        # eliminar directorio
$ rm -rf roles/backup/       # forzado, sin preguntar
⚠ Nunca uses: rm -rf / ni rm -rf /* — borraría todo el sistema operativo. Verifica siempre la ruta antes de ejecutar rm -rf.

permisos de archivos

Formato de permisos
rwxrwxrwx
dueño | grupo | otros
Valores numéricos
r=4, w=2, x=1
rwx = 4+2+1 = 7
chmod
chmod
Change Mode — cambiar permisos
permisos ▶

Define quién puede leer (r=4), escribir (w=2) o ejecutar (x=1) un archivo. Ansible requiere permisos específicos para funcionar correctamente.

uso con números
$ chmod 644 ansible.cfg         # rw-r--r--
$ chmod 755 playbooks/          # rwxr-xr-x
$ chmod 600 ~/.ssh/id_rsa       # rw------- (clave SSH)
$ chmod -R 644 playbooks/      # aplica a todo el dir
aplicar a proyecto completo
# directorios: 755
$ find ~/ansible-lab -type d -exec chmod 755 {} \;

# archivos: 644
$ find ~/ansible-lab -type f -exec chmod 644 {} \;
chown
chown
Change Owner — cambiar propietario
permisos ▶

Cambia el usuario y/o grupo dueño de un archivo o directorio.

ejemplos
$ sudo chown seba:seba archivo.yml
$ sudo chown -R seba:seba ~/ansible-lab/
$ sudo chown root:ansible /etc/ansible/ansible.cfg

tabla de permisos comunes

NúmeroSímboloUso típico
700rwx------Directorio privado del dueño
755rwxr-xr-xCarpetas del proyecto Ansible
644rw-r--r--Archivos de configuración / YAML
600rw-------Claves SSH privadas (~/.ssh/id_rsa)
400r--------Vault de Ansible (solo lectura)
775rwxrwxr-xCarpetas compartidas con grupo
Ansible requiere: directorios en 755, playbooks en 644, claves SSH en 600. Si los permisos son incorrectos, Ansible fallará con error de autenticación.

práctica guiada

01
Crear la estructura base del proyecto Ansible
Comandos: mkdir -p, tree, ls
1 Ir a tu directorio home:
bash
$ cd ~
$ pwd
2 Crear toda la estructura en un solo comando:
bash
$ mkdir -p ~/ansible-lab/{inventario,playbooks,roles,group_vars,host_vars}
3 Crear archivos base vacíos:
bash
$ touch ~/ansible-lab/inventario/hosts
$ touch ~/ansible-lab/playbooks/site.yml
$ touch ~/ansible-lab/ansible.cfg
4 Verificar la estructura:
bash
$ tree ~/ansible-lab
✓ VERIFICAR
comando de verificación
$ ls ~/ansible-lab | wc -l
5   ← debe mostrar 5 (cinco carpetas + ansible.cfg)
02
Crear el inventario de hosts
Comandos: cat heredoc, echo, grep
1 Escribir el inventario con heredoc:
bash
$ cat > ~/ansible-lab/inventario/hosts << 'EOF'
[webservers]
192.168.1.10
192.168.1.11

[dbservers]
192.168.1.20

[all:vars]
ansible_user=admin
ansible_python_interpreter=/usr/bin/python3
EOF
2 Leer el archivo creado:
bash
$ cat ~/ansible-lab/inventario/hosts
3 Hacer un backup del inventario:
bash
$ cp ~/ansible-lab/inventario/hosts ~/ansible-lab/inventario/hosts.bak
✓ VERIFICAR
contar IPs en el inventario
$ grep -c "192.168" ~/ansible-lab/inventario/hosts
3   ← debe mostrar 3
03
Aplicar permisos correctos al proyecto
Comandos: chmod, find, stat
1 Aplicar 755 a todos los directorios:
bash
$ find ~/ansible-lab -type d -exec chmod 755 {} \;
2 Aplicar 644 a todos los archivos:
bash
$ find ~/ansible-lab -type f -exec chmod 644 {} \;
3 Verificar permisos con ls:
bash
$ ls -la ~/ansible-lab/inventario/
✓ VERIFICAR
ver permisos exactos del archivo hosts
$ stat -c "%a %n" ~/ansible-lab/inventario/hosts
644 /home/seba/ansible-lab/inventario/hosts
04
Desafío final: estructura completa con roles
Todos los comandos vistos — estructura lista para producción
1 Crear la estructura completa del rol webserver:
bash
$ mkdir -p ~/ansible-lab/roles/webserver/{tasks,handlers,templates,files,vars,defaults,meta}
$ touch ~/ansible-lab/roles/webserver/tasks/main.yml
$ touch ~/ansible-lab/roles/webserver/handlers/main.yml
$ touch ~/ansible-lab/roles/webserver/defaults/main.yml
$ touch ~/ansible-lab/roles/webserver/meta/main.yml
2 Crear el playbook principal:
bash
$ cat > ~/ansible-lab/playbooks/site.yml << 'EOF'
---
- name: Configurar servidores web
  hosts: webservers
  roles:
    - webserver
EOF
3 Ver la estructura final:
bash
$ tree ~/ansible-lab
✓ RESULTADO ESPERADO
contar directorios creados
$ find ~/ansible-lab -type d | wc -l
13  ← debe mostrar 13 o más directorios

Configuración base del proyecto
config ▶

Archivo de configuración principal de Ansible. Colócalo en la raíz de tu proyecto.

~/ansible-lab/ansible.cfg
# ansible.cfg
[defaults]
inventory             = inventario/hosts
remote_user           = admin
host_key_checking     = False
retry_files_enabled   = False
roles_path            = roles

[privilege_escalation]
become                = True
become_method         = sudo
become_ask_pass       = False

Playbook — site.yml
Primer playbook de ejemplo
playbook ▶
playbooks/site.yml
---
# Playbook principal del proyecto

- name: Configurar servidores web
  hosts: webservers
  roles:
    - webserver

- name: Instalar base de datos
  hosts: dbservers
  tasks:
    - name: Instalar MariaDB
      ansible.builtin.dnf:
        name: mariadb-server
        state: present
run
Comandos para ejecutar Ansible
Los que más usarás cada día
ejecución ▶
comandos esenciales
# Verificar conectividad con los hosts
$ ansible all -m ping

# Ver el inventario en formato YAML
$ ansible-inventory --list -y

# Simulación — dry run (no hace cambios)
$ ansible-playbook playbooks/site.yml --check

# Ejecutar el playbook
$ ansible-playbook playbooks/site.yml

# Ejecutar con más detalle (verbose)
$ ansible-playbook playbooks/site.yml -v

# Solo en un grupo específico
$ ansible-playbook playbooks/site.yml --limit webservers
